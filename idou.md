# idou 架构实施清单 (Edge SSR & HTMX & Data Gateway Defense)

本清单用于规范 `idou` 数据网关链路的开发与部署。系统采用 **首屏 SSR（服务端渲染）结合 盘中 HTMX 局部响应** 的边缘闭环架构。在死守“零公开 JSON 接口、数据完全内聚、F12 隐匿交易特征”安全底线的同时，全面拉升盘中高频交互时的响应速度与算力经济性。

## 1. 基础设施与边缘拓扑优化 (Topology)
- [ ] **动静架构分离（官方规范）**：
  - [ ] 确保 `functions/` 目录严格置于项目根目录，作为核心页面与局部刷新的 Hono 编译引擎。
  - [ ] 彻底删除将 `functions/` 复制到 `dist/` 的构建逻辑。确保执行 `pnpm run build` 后，`dist/` 目录极其纯净，仅包含静态 HTML、CSS、`htmx.min.js`（或 CDN 引用），由 CDN 节点直发，严禁混入后端源码。
- [ ] **路由分流隔离**：已配置生产环境 `_routes.json`，确保非 API 的静态资源请求由 CDN 节点无损直发，防止静态资源意外唤醒 Worker 产生算力计费。
- [ ] **多环境隔离验证**：确保 `wrangler.toml` 包含正确的 `compatibility_date`，且 `[vars]` 在 Preview 与 Production 环境下已精准映射不同的上游数据源接口。

## 2. 治理层与极限防刷对抗 (Security & Resilience)
- [ ] **边缘级指纹限流 (Advanced WAF)**：
  - [ ] 开启 Cloudflare 边缘 WAF，基于客户端 **TLS 指纹（JA4+ 指纹）与 HTTP/2 指纹**进行特征识别，在触达 Hono 后端前，直接在边缘层阻断恶意爬虫。
  - [ ] 针对高频刷新、切换板块等行为，基于 IP 与指纹的双重维度设置高频熔断阈值。
- [ ] **动态会话鉴权**：在 Hono 根路由与 HTMX 内部渲染路由（如 `/internal-render/*`）统一挂载鉴权中间件，严格校验 `HX-Request` 头部标识。
- [ ] **静默蜜罐降级 (Silent Fallback)**：当遭遇签名异常、鉴权失败或指纹撞击时，后端**绝不抛出 403/500 等 HTTP 错误码**，而是统一返回无害的“伪装 HTML 碎片”或“空壳展示数据”（如泛灰的数据占位符 `<tr><td colspan="5" style="color:#cccccc">数据加载中...</td></tr>`），直接破坏反向工程的反馈回路。

## 3. 请求转发与数据内聚 (Proxy & Transform)
- [ ] **内网数据闭环**：Hono 在边缘端直接调用上游数据接口，所有交易数据流在 Cloudflare 内网闭环，绝不流经浏览器。
- [ ] **服务端数据清洗 (Transform)**：在上游响应到达 Hono 后，立即调用 `transform.js` 进行过滤、去噪与字段重命名，过滤一切非 UI 必需的 Metadata。
- [ ] **返回数据语义模糊化**：输出给 HTMX 渲染的内容必须经过二次脱敏，关键指标通过标签属性或噪点稀释（如使用混淆样式替代明文数字），提高对方自动化解析的成本。

## 4. 服务端渲染与零前端逻辑约束 (Thin-Client Logic)
- [ ] **核心数据全下沉**：所有核心行情数据的解析、清洗与组装全部在 Hono 服务端执行完毕，禁止在前端留下任何中间层处理痕迹。
- [ ] **UI 样式服务端着色**：核心视觉状态（如主板与创业板涨停红 `#D22222`、创业板 20% 涨停紫 `#800080`、蜜罐降级灰 `#cccccc`）必须在服务端通过 `hono/jsx` 完成模板拼接，直接输出带有内联样式的 HTML 标签。**（注意：严禁改动或污染原有的独立 CSS 文件）**。
- [ ] **前端纯净化（零自写 JS 交互）**：客户端浏览器仅通过 HTMX 声明式属性（如 `hx-get`、`hx-target`）触发局部刷新与 DOM 替换，前端无任何处理数据的 `if/else` 条件状态判断。

## 5. F12 源代码防窥与逆向阻断 (Anti-Reverse Engineering)
- [ ] **关闭 Source Map**：构建脚本中完全禁用 Source Map (`sourcemap: false`)，彻底封死前端目录树还原路径。
- [ ] **网络面板隐形审查**：经 F12 现场审计，Network（网络）选项卡中**完全不存在任何结构化的 JSON 数据包**。HTMX 触发的异步请求返回的全部是已经拼装好样式的 HTML 碎片。
- [ ] **DOM 结构加固与反调试**：
  - [ ] 页面注入混淆后的防调试逻辑，通过定时触发 `debugger` 阻止控制台审查，拦截鼠标右键与 F12 键。
  - [ ] 关键数据节点（如股票代码、指标标签）在 HTML 碎片中混入隐藏的噪点文本，阻断简单的 `html.parser` 自动化文本抓取。

## 6. 环境隔离与交付验证 (Isolation & Verification)
- [ ] **敏感信息零硬编码**：确认无任何 Token 或 API 路径硬编码在代码中，全部存于 Cloudflare 环境变量或 Secrets 中。
- [ ] **自动化 GitOps**：GitHub 代码提交无感自动触发 Cloudflare Pages 的构建与全量边缘部署。
- [ ] **极限黑盒审计**：使用 `curl` 及 Python 脚本直接撞击页面及 HTMX 局部路由，确认未携带合法凭证及指纹时，系统仅返回蜜罐伪装页面的 HTML 源码。
